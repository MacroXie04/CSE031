C/C++ 系统学习大纲（UC CSE 专业导向）
第一阶段：C 语言基础（约 4–6 周）
1. 开发环境与编译 熟悉 GCC/Clang 编译器、Makefile 的基本写法，以及 GDB 调试器的使用。UC 的课程普遍在 Linux/Unix 环境下完成作业，所以尽早习惯终端操作。

2. 数据类型与运算 掌握基本类型（int, char, float, double）、类型转换规则、sizeof 运算符，以及整型溢出、浮点精度等常见陷阱。理解 unsigned 与 signed 的区别。

3. 控制流与函数 条件语句、循环、switch-case，以及函数的声明与定义、参数传递（值传递）、递归。理解函数调用栈的概念。

4. 指针（核心重点） 这是 C 语言最关键的部分，也是 UC CSE 课程反复考察的内容：指针的声明与解引用、指针算术、指针与数组的关系、多级指针、函数指针、void 指针。一定要能画出内存图来理解指针操作。

5. 数组与字符串 一维/多维数组、数组作为函数参数的退化行为、C 风格字符串（char 数组 + '\0'）、常用字符串函数（strlen, strcpy, strcmp, strcat）及其安全版本（strncpy 等）。

6. 内存管理 理解栈（stack）与堆（heap）的区别，malloc/calloc/realloc/free 的使用，内存泄漏与悬挂指针的检测（Valgrind 工具）。这在 UC 的系统编程课中极为重要。

7. 结构体与自定义类型 struct 的定义与使用、结构体指针与箭头运算符、typedef、枚举（enum）、联合体（union）、位域。

8. 文件 I/O 与预处理器 fopen/fclose/fread/fwrite/fprintf/fscanf，以及宏定义（#define）、条件编译（#ifdef）、头文件的组织与 include guard。

9. 用 C 实现基础数据结构 链表（单链表、双链表）、栈、队列、哈希表、二叉树。这些在 UC 的数据结构课（如 CSE 100）中会反复用到，用 C 实现一遍能极大加深对指针和内存的理解。

第二阶段：C++ 基础与面向对象（约 4–6 周）
1. 从 C 到 C++ 的过渡 C++ 对 C 的兼容与扩展、命名空间（namespace）、引用（reference）vs 指针、cin/cout 与流、new/delete vs malloc/free、bool 类型。

2. 类与对象 类的定义、访问控制（public/private/protected）、构造函数与析构函数、拷贝构造函数、赋值运算符重载、this 指针、const 成员函数。

3. 三/五法则（Rule of Three / Five） 深拷贝与浅拷贝问题、析构函数 + 拷贝构造 + 拷贝赋值（Rule of Three）；移动构造 + 移动赋值（Rule of Five，C++11）。这是 UC 课程中常考的设计题。

4. 运算符重载 重载算术运算符、比较运算符、流插入/提取运算符（<< / >>）、下标运算符等。理解成员函数 vs 友元函数的选择。

5. 继承与多态 单继承与多继承、虚函数（virtual）与动态绑定、纯虚函数与抽象类、虚析构函数（为什么必须有）、vtable 的基本原理、向上转型与向下转型（dynamic_cast）。

6. 模板（Templates） 函数模板与类模板、模板特化、模板参数推导。理解 STL 容器为什么能工作的底层原理。

第三阶段：C++ 进阶与现代特性（约 3–4 周）
1. STL（标准模板库） 容器：vector, list, deque, map, unordered_map, set, stack, queue, priority_queue。迭代器的分类与使用。算法：sort, find, transform, accumulate, lower_bound 等。理解时间复杂度。

2. 智能指针与现代内存管理 unique_ptr、shared_ptr、weak_ptr 的语义与使用场景，RAII（Resource Acquisition Is Initialization）原则。

3. C++11/14/17 重要特性 auto 类型推导、范围 for 循环、lambda 表达式、右值引用与移动语义（std::move）、std::optional、结构化绑定、constexpr。

# CSE 31 — Computer Organization: Comprehensive Study Outline

> **Course Info**: UC Merced · Spring 2026 · Instructors: Santosh Chandrasekhar & Hamid Rajabi
> **Textbook**: zyBooks — Computer Organization and Assembly Language
> **Supplementary**: K&R — *C Programming Language, 2nd Edition*
> **Exams**: Midterm 1 (Week 8) · Midterm 2 (Week 13) · Final (5/14)

---

## Part 1: Number Systems & Data Representation

> 🎯 Course Objective 1: Describe how data are represented in the computer, including floating point numbers and arrays.

---

### Module 1: Number Systems

#### Unit 1.1: Base Systems
- **Key Concepts**
    - Decimal (base-10) positional notation: $d_n \times 10^n + ... + d_0 \times 10^0$
    - Binary (base-2): bit, byte (= 8 bits), word (= 32 bits in MIPS)
    - Hexadecimal (base-16): digits 0–9 and A–F mapping
    - Octal (base-8): awareness level, rarely tested

#### Unit 1.2: Base Conversions
- **Key Concepts**
    - Decimal → Binary: repeated division by 2 (integer part), repeated multiplication by 2 (fractional part)
    - Binary → Decimal: positional weight expansion and summation
    - Binary ↔ Hexadecimal: group every 4 binary digits = 1 hex digit (fast conversion)
    - Decimal → Hexadecimal: divide by 16, or convert to binary first then group
    - Fractional base conversion and precision loss (e.g., 0.1₁₀ cannot be exactly represented in binary)

#### Unit 1.3: Binary Arithmetic
- **Key Concepts**
    - Binary addition: bit-by-bit with carry
    - Binary subtraction: bit-by-bit with borrow
    - Binary multiplication: shift-and-add method
    - Overflow: result exceeds representable range
    - Bitwise operations: AND (&), OR (|), XOR (^), NOT (~), left shift (<<), right shift (>>)
    - Applications of bitwise ops: masking, setting/clearing/toggling specific bits

---

### Module 2: Integer Representation

#### Unit 2.1: Unsigned Integers
- **Key Concepts**
    - Range of n-bit unsigned integer: $0$ to $2^n - 1$
    - 8-bit range: 0–255
    - 16-bit range: 0–65,535
    - 32-bit range: 0–4,294,967,295
    - Unsigned overflow: wraparound behavior

#### Unit 2.2: Sign-Magnitude
- **Key Concepts**
    - Most Significant Bit (MSB) is the sign bit: 0 = positive, 1 = negative
    - n-bit range: $-(2^{n-1} - 1)$ to $+(2^{n-1} - 1)$
    - The +0 / -0 problem (two representations of zero)
    - Complexity of addition/subtraction (need to compare magnitudes)
    - Why sign-magnitude is not used in modern computers

#### Unit 2.3: One's Complement
- **Key Concepts**
    - Positive numbers: same as sign-magnitude
    - Negative numbers: flip all bits (0↔1)
    - n-bit range: $-(2^{n-1} - 1)$ to $+(2^{n-1} - 1)$
    - Still has +0 / -0 problem
    - Addition rule: end-around carry (add the carry-out back to the result)
    - Historical significance (used in early computers)

#### Unit 2.4: Two's Complement ⭐ Critical Topic
- **Key Concepts**
    - Positive numbers: same as binary
    - Negative numbers: flip all bits + add 1 (i.e., one's complement + 1)
    - Shortcut: scan from right, find first 1, flip everything to the left of it
    - n-bit range: $-2^{n-1}$ to $+2^{n-1} - 1$ (asymmetric)
    - 8-bit range: -128 to +127
    - 32-bit range: -2,147,483,648 to +2,147,483,647
    - Only one representation of zero (solves the double-zero problem)
    - Most negative value $-2^{n-1}$ has no positive counterpart
    - Two's complement addition: add directly, discard carry-out from MSB
    - Two's complement subtraction: $A - B = A + (-B)$, where $-B$ is the two's complement of $B$
    - **Overflow detection**: two positives yield negative, or two negatives yield positive
    - **Sign extension**: extending n bits to m bits (m > n) by copying the sign bit into the upper bits
    - **Zero extension**: for unsigned numbers, fill upper bits with 0
    - Why two's complement is the standard: single adder handles both add/subtract, only one zero

#### Unit 2.5: Bias Encoding
- **Key Concepts**
    - Bias concept: stored value = actual value + bias
    - Used for the exponent field in IEEE 754 floating point
    - Purpose: allows exponent comparison using unsigned comparison
    - Converting between biased and actual values given a specific bias

---

### Module 3: Floating-Point Representation

#### Unit 3.1: IEEE 754 Standard ⭐ Critical Topic
- **Key Concepts**
    - Floating-point scientific notation form: $(-1)^S \times 1.M \times 2^E$
    - **Single precision (float, 32-bit) fields**:
        - Sign (S): 1 bit (0 = positive, 1 = negative)
        - Exponent (E): 8 bits, bias = 127
        - Mantissa / Fraction (M): 23 bits
    - **Double precision (double, 64-bit) fields**:
        - Sign: 1 bit
        - Exponent: 11 bits, bias = 1023
        - Mantissa: 52 bits
    - Implicit leading 1: value = $1.M$, so actual precision is one bit more than stored
    - True exponent = stored exponent - bias

#### Unit 3.2: Special Values
- **Key Concepts**
    - **Normalized numbers**: exponent ≠ all 0s and ≠ all 1s
    - **Denormalized numbers (Subnormal)**: exponent = all 0s, mantissa ≠ 0
        - Implicit leading bit becomes 0: $0.M \times 2^{1-\text{bias}}$
        - Enables gradual underflow (representing very small numbers near zero)
    - **Zero**: exponent = all 0s, mantissa = all 0s (+0 and -0 exist)
    - **Infinity**: exponent = all 1s, mantissa = all 0s (+∞ and -∞)
    - **NaN (Not a Number)**: exponent = all 1s, mantissa ≠ 0
        - Produced by: 0/0, ∞ - ∞, √(-1), etc.

#### Unit 3.3: Decimal ↔ IEEE 754 Conversion ⭐ Guaranteed Exam Topic
- **Key Concepts**
    - **Decimal → IEEE 754 steps**:
        1. Determine the sign bit
        2. Convert the absolute value to binary
        3. Normalize to $1.xxxx \times 2^n$ form
        4. Compute stored exponent = $n$ + bias
        5. Fill the fractional part into the mantissa field
    - **IEEE 754 → Decimal steps**:
        1. Extract sign bit, exponent field, mantissa field
        2. Compute true exponent = stored exponent - bias
        3. Reconstruct significand: $1.\text{mantissa}$ (normalized) or $0.\text{mantissa}$ (denormalized)
        4. Compute value: $(-1)^S \times \text{significand} \times 2^{\text{true exponent}}$
    - **Practice**: be able to do single-precision conversions both ways by hand

#### Unit 3.4: Precision and Limitations
- **Key Concepts**
    - Precision limits: single ≈ 7 significant decimal digits, double ≈ 15
    - Range: single ≈ $\pm 3.4 \times 10^{38}$
    - Rounding error
    - Some decimal fractions cannot be represented exactly (e.g., 0.1)
    - Never compare floats with `==`; use an epsilon tolerance
    - Floating-point addition is not associative: $(a + b) + c \neq a + (b + c)$ in some cases

## Part 2: C Programming

> 🎯 Course Objective 2: Demonstrate how fundamental high-level programming constructs are implemented at the machine-language level.

---

### Module 4: C Language Fundamentals

#### Unit 4.1: Program Structure and Compilation
- **Key Concepts**
    - Basic C program structure: `#include`, `main` function
    - `printf` format specifiers: `%d`, `%u`, `%x`, `%f`, `%lf`, `%c`, `%s`, `%p`, `%ld`, `%lu`
    - `scanf` formatted input: requires `&` for address (except strings)
    - `printf` / `scanf` format modifiers: width, precision, alignment
    - Compilation command: `gcc -o output source.c`
    - Four stages of compilation: Preprocessing → Compilation → Assembly → Linking
    - Common compiler flags: `-Wall`, `-Werror`, `-g`, `-std=c11`, `-O0`/`-O2`
    - Header files vs source files
    - `#include <stdio.h>` vs `#include "myheader.h"`

#### Unit 4.2: Data Types and Variables
- **Key Concepts**
    - Primitive types: `char`(1B), `short`(2B), `int`(4B), `long`(4/8B), `long long`(8B)
    - `signed` vs `unsigned` differences and value ranges
    - `float`(4B) vs `double`(8B)
    - `sizeof` operator
    - Variable declaration, initialization, scope
    - Type conversion: implicit (integer promotion, arithmetic conversion) and explicit (casting)
    - `const` constants
    - `#define` macro constants

#### Unit 4.3: Operators and Expressions
- **Key Concepts**
    - Arithmetic: `+`, `-`, `*`, `/` (integer division truncation), `%`
    - Relational and logical: `==`, `!=`, `<`, `>`, `&&`, `||`, `!`
    - Bitwise: `&`, `|`, `^`, `~`, `<<`, `>>` (⭐ tested together with data representation)
    - Assignment: `=`, `+=`, `-=`, `<<=`, `>>=`, `&=`, `|=`, `^=`
    - Increment/decrement: `++i` vs `i++`
    - Operator precedence (memorize the common ones)
    - Short-circuit evaluation

#### Unit 4.4: Control Flow
- **Key Concepts**
    - `if` / `else if` / `else`
    - `switch` / `case` / `default` / `break` / fall-through behavior
    - `while` loop
    - `do-while` loop
    - `for` loop
    - `break` and `continue`
    - Nested loops
    - How these control-flow constructs map to assembly (preview for MIPS modules)

#### Unit 4.5: Functions
- **Key Concepts**
    - Function declaration (prototype) vs definition
    - Parameter passing: pass by value
    - Return values
    - Call stack concept: stack frame, return address, local variables
    - Recursive function call stack analysis
    - `main(int argc, char *argv[])` command-line arguments

### Module 5: Pointers ⭐ Core Topic

#### Unit 5.1: Pointer Basics
- **Key Concepts**
    - Concept of memory addresses
    - Pointer variable declaration: `int *p;`
    - Address-of operator `&`: `p = &x;`
    - Dereference operator `*`: `*p = 10;`
    - Pointer size (32-bit system = 4 bytes, 64-bit = 8 bytes; MIPS is 32-bit)
    - `NULL` pointer
    - Wild pointers and dangling pointers
    - ⭐ **Draw memory diagrams** to understand pointer operations (do this for every pointer problem)

#### Unit 5.2: Pointer Arithmetic ⭐ Course Emphasis
- **Key Concepts**
    - Pointer addition: `p + n` equals address `p + n * sizeof(*p)`
    - Pointer subtraction: difference of two pointers = number of elements between them
    - Pointer comparison: `<`, `>`, `==`, `!=`
    - Pointer increment/decrement: `p++`, `p--`
    - Differences between `*p++`, `(*p)++`, and `*++p` (⭐ frequently tested)
    - Relationship between pointer arithmetic and array traversal
    - Different pointer types have different step sizes: `int *` steps by 4, `char *` steps by 1

#### Unit 5.3: Pointers and Arrays
- **Key Concepts**
    - Array name ≈ pointer to first element (but not exactly equivalent)
    - `a[i]` is equivalent to `*(a + i)`
    - Arrays decay to pointers when passed to functions
    - `sizeof(array)` vs `sizeof(pointer)` difference
    - Traversing arrays with pointers vs with subscripts

#### Unit 5.4: Pointers and Functions
- **Key Concepts**
    - Simulating pass-by-reference using pointers
    - Classic example: `void swap(int *a, int *b)`
    - Functions returning pointers (never return address of a local variable)
    - Pointer-to-pointer (`int **pp`)

#### Unit 5.5: const and Pointers
- **Key Concepts**
    - `const int *p`: pointer to constant (value cannot be changed through p)
    - `int * const p`: constant pointer (p itself cannot be reassigned)
    - `const int * const p`: both are const
    - Role of const pointers in function parameters (protecting data from modification)

---

### Module 6: Arrays & C Strings

#### Unit 6.1: One-Dimensional Arrays
- **Key Concepts**
    - Array declaration and initialization: `int arr[5] = {1, 2, 3, 4, 5};`
    - Partial initialization: `int arr[5] = {1, 2};` (remaining elements are 0)
    - Out-of-bounds access: C does not perform bounds checking → undefined behavior
    - Contiguous memory storage of arrays
    - Calculating array length: `sizeof(arr) / sizeof(arr[0])`

#### Unit 6.2: Multi-Dimensional Arrays
- **Key Concepts**
    - 2D array declaration: `int matrix[3][4];`
    - Row-major memory layout
    - Element address calculation: `base + (row × cols + col) × sizeof(element)`
    - 2D arrays as function parameters: `void func(int arr[][4], int rows)`

#### Unit 6.3: C Strings ⭐ Course Emphasis
- **Key Concepts**
    - C string = `char` array + null terminator `'\0'`
    - String literals: `"hello"` stored in read-only data segment
    - `char str[] = "hello"` vs `char *str = "hello"` — fundamental difference:
        - Former: allocates 6 bytes on the stack, modifiable
        - Latter: `str` points to read-only data segment, not modifiable
    - String length `strlen(s)` vs array size `sizeof(str)`
    - Standard string functions (`<string.h>`):
        - `strlen(s)` — returns length (excluding `'\0'`)
        - `strcpy(dest, src)` / `strncpy(dest, src, n)` — copy
        - `strcat(dest, src)` / `strncat(dest, src, n)` — concatenation
        - `strcmp(s1, s2)` — comparison (return value meaning: <0, 0, >0)
        - `strchr(s, c)` / `strstr(s1, s2)` — search
    - Buffer overflow in string operations
    - Manually implementing `strlen`, `strcpy`, etc. (deepens understanding of pointers and strings)

---

### Module 7: Structs

#### Unit 7.1: Struct Basics
- **Key Concepts**
    - `struct` definition and declaration
    - Struct initialization: `{}`, C99 designated initializers `.field = value`
    - Member access: `.` operator
    - Struct assignment (memberwise copy / shallow copy)
    - `typedef struct { ... } TypeName;` shorthand

#### Unit 7.2: Structs and Memory
- **Key Concepts**
    - Struct memory layout
    - **Memory alignment and padding** ⭐
        - Each member's address must be a multiple of its size
        - Total struct size must be a multiple of the largest member's size
    - Calculating actual struct size (accounting for padding)
    - `sizeof(struct)` may be larger than sum of member sizes
    - Effect of member ordering on struct size

#### Unit 7.3: Structs and Pointers
- **Key Concepts**
    - Struct pointer: `struct Node *p`
    - Arrow operator: `p->field` is equivalent to `(*p).field`
    - Dynamically allocating structs: `malloc(sizeof(struct Node))`
    - Structs as function parameters: pass by value vs pass by pointer
    - Self-referential structs (linked list node definition)

---

### Module 8: Memory Management ⭐

#### Unit 8.1: Program Memory Layout
- **Key Concepts**
    - **Text / Code Segment**: stores machine instructions, read-only
    - **Data Segment**: initialized global/static variables
    - **BSS Segment**: uninitialized global/static variables (zero-initialized)
    - **Heap**: dynamic allocation, grows toward higher addresses
    - **Stack**: local variables, function call info, grows toward lower addresses
    - Memory layout diagram (⭐ must be able to draw this)

#### Unit 8.2: Dynamic Memory Allocation
- **Key Concepts**
    - `malloc(size)` — allocates `size` bytes of uninitialized memory
    - `calloc(count, size)` — allocates and zero-initializes

### Module 11: MIPS Architecture Overview

#### Unit 11.1: MIPS Basics
- **Key Concepts**
    - MIPS = Microprocessor without Interlocked Pipeline Stages
    - RISC (Reduced Instruction Set Computer): fixed-length instructions (32 bits), load-store architecture
    - RISC vs CISC differences
    - Word = 32 bits = 4 bytes
    - Byte-addressable memory and word alignment (address must be a multiple of 4)
    - Big Endian vs Little Endian byte ordering
    - MIPS simulator tools: MARS or QtSpim

#### Unit 11.2: MIPS Registers ⭐ Must Memorize
- **Key Concepts**
    - MIPS has 32 general-purpose registers ($0 – $31), each 32 bits wide
    - Important registers and their conventional uses:

  | Register | Number | Purpose |
    |----------|--------|---------|
  | `$zero` | $0 | Always 0 |
  | `$at` | $1 | Assembler temporary |
  | `$v0–$v1` | $2–$3 | Function return values / syscall number |
  | `$a0–$a3` | $4–$7 | Function arguments (first 4) |
  | `$t0–$t9` | $8–$15, $24–$25 | Temporaries (caller-saved) |
  | `$s0–$s7` | $16–$23 | Saved registers (callee-saved) |
  | `$gp` | $28 | Global pointer |
  | `$sp` | $29 | Stack pointer |
  | `$fp` | $30 | Frame pointer |
  | `$ra` | $31 | Return address |

    - Special registers: `HI` and `LO` (multiply/divide results), `PC` (Program Counter)
    - Caller-saved vs callee-saved register conventions

### Module 12: MIPS Instruction Set

#### Unit 12.1: Arithmetic and Logic Instructions
- **Key Concepts**
    - **R-type instructions** (register-register operations):
        - `add $d, $s, $t` — $d = $s + $t
        - `sub $d, $s, $t` — $d = $s - $t
        - `and $d, $s, $t` / `or $d, $s, $t` / `xor $d, $s, $t`
        - `nor $d, $s, $t` — NOR operation (NOT can be done with `nor $d, $s, $zero`)
        - `sll $d, $t, shamt` — shift left logical
        - `srl $d, $t, shamt` — shift right logical
        - `sra $d, $t, shamt` — shift right arithmetic (preserves sign bit)
        - `slt $d, $s, $t` — set on less than ($d = ($s < $t) ? 1 : 0, signed)
        - `sltu $d, $s, $t` — unsigned version
    - **I-type instructions** (register-immediate operations):
        - `addi $t, $s, imm` — $t = $s + imm (16-bit signed immediate)
        - `addiu $t, $s, imm` — unsigned add (no overflow exception)
        - `andi $t, $s, imm` / `ori $t, $s, imm` / `xori $t, $s, imm`
        - `slti $t, $s, imm` / `sltiu $t, $s, imm`
        - `lui $t, imm` — load upper immediate (loads immediate into upper 16 bits)
    - Multiply and divide:
        - `mult $s, $t` — result stored in HI:LO
        - `div $s, $t` — LO = quotient, HI = remainder
        - `mfhi $d` / `mflo $d` — move from HI/LO

#### Unit 12.2: Data Transfer Instructions (Load/Store)
- **Key Concepts**
    - MIPS is a load-store architecture: arithmetic only operates on registers
    - `lw $t, offset($s)` — load word: $t = Memory[$s + offset] (4 bytes)
    - `sw $t, offset($s)` — store word: Memory[$s + offset] = $t (4 bytes)
    - `lb $t, offset($s)` — load byte (sign-extended)
    - `lbu $t, offset($s)` — load byte unsigned (zero-extended)
    - `sb $t, offset($s)` — store byte
    - `lh` / `lhu` / `sh` — halfword (2-byte) operations
    - Word alignment requirement: `lw`/`sw` addresses must be multiples of 4
    - Loading 32-bit constants: `lui` + `ori` combination
        - Example: load `0x12345678` → `lui $t0, 0x1234` then `ori $t0, $t0, 0x5678`
    - Pseudo-instructions: `li $t0, value` and `la $t0, label`

#### Unit 12.3: Branch and Jump Instructions
- **Key Concepts**
    - Conditional branches (I-type):
        - `beq $s, $t, label` — branch if equal
        - `bne $s, $t, label` — branch if not equal
    - Unconditional jumps (J-type):
        - `j label` — jump
        - `jal label` — jump and link (stores PC+4 into $ra, used for function calls)
        - `jr $ra` — jump to register address (used for function returns)
    - Branch address calculation: PC + 4 + offset × 4 (PC-relative addressing)
    - Jump address calculation: $\{PC[31:28], \text{address}, 00\}$ (pseudodirect addressing)
    - Common pseudo-instructions:
        - `blt $s, $t, label` — branch if less than (translates to slt + bne)
        - `bgt`, `ble`, `bge`, etc.

#### Unit 12.4: System Calls (syscall)
- **Key Concepts**
    - MIPS uses `syscall` for I/O and program control
    - Common syscall numbers (value in $v0):

  | $v0 | Function | Arguments |
    |-----|----------|-----------|
  | 1 | Print integer | $a0 = integer |
  | 2 | Print float | $f12 = float |
  | 4 | Print string | $a0 = string address |
  | 5 | Read integer | Result in $v0 |
  | 8 | Read string | $a0 = buffer address, $a1 = max length |
  | 9 | Allocate heap memory (sbrk) | $a0 = byte count, result in $v0 |
  | 10 | Exit program | — |
  | 11 | Print character | $a0 = character |
  | 12 | Read character | Result in $v0 |

### Module 13: MIPS Programming Patterns

#### Unit 13.1: C → MIPS Translation ⭐ Core Exam Topic
- **Key Concepts**
    - **Translating if-else statements**:
      ```
      # if (a == b) { ... } else { ... }
      bne $s0, $s1, else_label
      # if body
      j end_if
      else_label:
      # else body
      end_if:
      ```
    - **Translating while loops**:
      ```
      loop:
      beq $s0, $zero, end_loop    # while (a != 0)
      # loop body
      j loop
      end_loop:
      ```
    - **Translating for loops**: initialize → check condition → loop body → update → jump back to condition
    - **Translating switch-case**: jump table or chained beq instructions
    - Translating compound conditions (`&&`, `||`)
    - ⭐ Practice: given a C code snippet, hand-write the equivalent MIPS assembly

#### Unit 13.2: Array Access Translation
- **Key Concepts**
    - Array base address + offset = element address
    - Translating `arr[i]`:
      ```
      # $s0 = base address of arr, $s1 = i
      sll $t0, $s1, 2        # $t0 = i * 4 (for int array)
      add $t0, $s0, $t0      # $t0 = &arr[i]
      lw  $t1, 0($t0)        # $t1 = arr[i]
      ```
    - Character arrays (char[]): offset does not need × 4 (use `lb`/`sb`)
    - MIPS implementation of array traversal

#### Unit 13.3: Function Calling Convention ⭐ Core Exam Topic
- **Key Concepts**
    - **Caller's responsibilities**:
        1. Place arguments in `$a0–$a3`
        2. Save caller-saved registers (`$t0–$t9`) if needed later
        3. Execute `jal function_label`
        4. Retrieve return value from `$v0`
        5. Restore caller-saved registers
    - **Callee's responsibilities**:
        1. Adjust stack pointer: `addi $sp, $sp, -N` (allocate stack frame)
        2. Save callee-saved registers (`$s0–$s7`) and `$ra` (if this function calls other functions)
        3. Execute function body
        4. Place return value in `$v0`
        5. Restore callee-saved registers and `$ra`
        6. Deallocate stack frame: `addi $sp, $sp, N`
        7. `jr $ra` to return
    - Stack frame structure diagram (⭐ must be able to draw this)
    - How `$sp` changes throughout execution
    - Leaf function vs non-leaf function
    - Non-leaf functions must save `$ra`

#### Unit 13.4: Recursion in MIPS
- **Key Concepts**
    - Implementing recursive functions in MIPS (factorial, Fibonacci, etc.)
    - Stack frame push/pop for each recursive call
    - Base case branch
    - Drawing the stack frame changes through recursive calls
    - ⭐ Must be able to hand-write complete MIPS code for recursive functions

### Module 14: MIPS Machine Code Encoding

#### Unit 14.1: Three Instruction Formats ⭐ Guaranteed Exam Topic
- **Key Concepts**
    - **R-type format** (6 + 5 + 5 + 5 + 5 + 6 = 32 bits):
      ```
      | opcode (6) | rs (5) | rt (5) | rd (5) | shamt (5) | funct (6) |
      ```
        - opcode = 0 for all R-type instructions
        - funct field differentiates operations (e.g., add=0x20, sub=0x22, and=0x24, or=0x25, slt=0x2A)
        - shamt used for shift instructions
    - **I-type format** (6 + 5 + 5 + 16 = 32 bits):
      ```
      | opcode (6) | rs (5) | rt (5) | immediate (16) |
      ```
        - 16-bit immediate, sign-extended (arithmetic) or zero-extended (logical)
        - Used for addi, lw, sw, beq, bne, etc.
    - **J-type format** (6 + 26 = 32 bits):
      ```
      | opcode (6) | address (26) |
      ```
        - Used for j, jal

#### Unit 14.2: Encoding and Decoding Instructions ⭐ Guaranteed Exam Topic
- **Key Concepts**
    - **Assembly → machine code encoding steps**:
        1. Determine instruction type (R/I/J)
        2. Look up opcode and funct values
        3. Convert register names to numbers
        4. Convert each field to binary
        5. Combine into 32-bit machine code
        6. Convert to hexadecimal representation
    - **Machine code → assembly decoding steps**:
        1. Extract opcode field (first 6 bits)
        2. Determine type from opcode
        3. If R-type (opcode=0), check funct field for specific operation
        4. Extract fields and convert back to register names and values
    - **Practice**: given assembly, write hex machine code; given machine code, write assembly
    - Common opcode table (need to memorize or use a reference table)

#### Unit 14.3: Addressing Modes
- **Key Concepts**
    - Register Addressing: operand is in a register (R-type)
    - Immediate Addressing: operand is a constant in the instruction (I-type)
    - Base Addressing: address = register + offset (lw, sw)
    - PC-Relative Addressing: branch target = PC + 4 + offset × 4 (beq, bne)
    - Pseudodirect Addressing: jump target = {PC[31:28], address, 00} (j, jal)

### Module 18: Cache ⭐ Core Topic

#### Unit 18.1: Cache Fundamentals
- **Key Concepts**
    - Cache Hit: requested data is in the cache
    - Cache Miss: requested data is not in the cache, must fetch from next level
    - Hit Rate = hits / total accesses
    - Miss Rate = 1 - Hit Rate
    - Hit Time: time to access the cache
    - Miss Penalty: additional time to fetch data from the next level
    - Average Memory Access Time (AMAT) = Hit Time + Miss Rate × Miss Penalty

#### Unit 18.2: Three Types of Cache Misses (The 3 C's) ⭐
- **Key Concepts**
    - **Compulsory Miss (Cold Miss)**:
        - First access to a piece of data; cache is guaranteed to not have it
        - Occurs even with an infinitely large cache
    - **Capacity Miss**:
        - Cache is too small to hold all needed data
        - Occurs even with a fully associative cache
    - **Conflict Miss (Collision Miss)**:
        - Multiple data blocks map to the same cache location and evict each other
        - Only occurs in direct-mapped and set-associative caches; not in fully associative

#### Unit 18.3: Direct-Mapped Cache
- **Key Concepts**
    - Each memory block maps to exactly one cache line
    - Address decomposition:
      ```
      | Tag | Index | Offset |
      ```
        - **Offset** (block offset): $\log_2(\text{block size})$ bits
        - **Index** (cache line number): $\log_2(\text{number of cache lines})$ bits
        - **Tag**: remaining bits
    - Valid bit: indicates whether the cache line holds valid data
    - Lookup process: use Index to find cache line → compare Tag → check Valid bit
    - Pros: fast lookup, simple hardware
    - Cons: high conflict misses (thrashing / ping-pong effect)

#### Unit 18.4: Fully Associative Cache
- **Key Concepts**
    - Any memory block can go in any cache line
    - Address decomposition:
      ```
      | Tag | Offset |
      ```
        - No Index field
    - Lookup process: compare Tag against all cache lines in parallel
    - Pros: no conflict misses
    - Cons: requires comparing all tags simultaneously, expensive hardware (many comparators)

#### Unit 18.5: N-Way Set Associative Cache ⭐ Exam Focus
- **Key Concepts**
    - Cache is divided into sets; each set has N ways (lines)
    - Each memory block maps to one set, can go in any way within that set
    - Address decomposition:
      ```
      | Tag | Set Index | Offset |
      ```
        - **Offset**: $\log_2(\text{block size})$ bits
        - **Set Index**: $\log_2(\text{number of sets})$ bits
        - **Tag**: remaining bits
    - Number of sets = total cache lines / N
    - Direct-mapped = 1-way; fully associative = (total lines)-way
    - Lookup process: use Set Index to find the set → compare Tag against all N ways in parallel
    - Trade-off: increasing N reduces conflict misses but increases hardware complexity and lookup time

#### Unit 18.6: Cache Address Calculation Practice ⭐ Guaranteed Exam Topic
- **Key Concepts**
    - Given cache parameters (total size, block size, associativity), calculate:
        - Number of sets = cache size / (block size × associativity)
        - Offset bits = $\log_2(\text{block size})$
        - Index bits = $\log_2(\text{number of sets})$
        - Tag bits = address bits - Index bits - Offset bits
    - Given a sequence of addresses, simulate cache accesses and determine hit/miss
    - Calculate hit rate

#### Unit 18.7: Replacement Policies
- **Key Concepts**
    - **LRU (Least Recently Used)**: evict the block that hasn't been used for the longest time
        - Requires tracking usage order
        - Hardware overhead grows with associativity
    - **FIFO (First In First Out)**: evict the oldest block
    - **Random**: randomly select a block to evict
    - **Approximate LRU**: Pseudo-LRU, clock algorithm
    - Performance comparison of different replacement policies

#### Unit 18.8: Write Policies
- **Key Concepts**
    - **Write Hit policies**:
        - Write-Through: update both cache and memory simultaneously
        - Write-Back: update only cache, mark with dirty bit, write to memory on eviction
    - **Write Miss policies**:
        - Write-Allocate: load the block into cache, then write
        - No-Write-Allocate (Write-Around): write directly to memory, skip cache
    - Common pairings: Write-Back + Write-Allocate, Write-Through + No-Write-Allocate
    - Write buffer: absorbs write-through traffic to avoid stalling the processor

### C. Learning Progress Checklist

**Part 1: Data Representation**
- [ ] Can quickly convert between binary ↔ hexadecimal ↔ decimal
- [ ] Can perform two's complement addition/subtraction by hand and detect overflow
- [ ] Can do IEEE 754 single-precision conversions in both directions on paper

**Part 2: C Programming**
- [ ] Can draw memory diagrams to analyze any pointer operation
- [ ] Can hand-write strlen, strcpy, and similar string functions
- [ ] Can calculate struct sizes (accounting for padding)
- [ ] Can correctly use malloc/free with no memory errors
- [ ] Can use GDB to locate a segfault

**Part 3: MIPS Assembly**
- [ ] Can name all 32 MIPS registers and their purposes from memory
- [ ] Can translate if/else, while, and for loops into MIPS
- [ ] Can hand-write complete MIPS programs with function calls (including recursion)
- [ ] Can encode MIPS instructions into 32-bit machine code (R/I/J types)

- [ ] Can draw the single-cycle MIPS datapath and label all control signals

- [ ] Can apply LRU replacement policy

**Part 6: Digital Logic**

- [ ] Can simplify Boolean expressions using De Morgan's Laws
