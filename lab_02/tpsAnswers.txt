TPS 2
======

1. Compile with the -g flag to include debugging information:
   gcc -g -o punish punishment.c

2. Load the program in LLDB (Mac):
   lldb punish
   (For GDB: gdb punish)

3. Run the program inside the debugger:
   LLDB: run (or r)
   GDB: run (or r)

4. Breakpoints are markers that tell the debugger to pause execution at a specific line.
   To set a breakpoint at a line:
   LLDB: breakpoint set --line <line_number>  (or b <line_number>)
   GDB: break <line_number>  (or b <line_number>)

5. To run line by line (step over):
   LLDB: next (or n)
   GDB: next (or n)
   To step into a function call: step (or s)

6. To see the value of a variable:
   LLDB: print <variable_name>  (or p <variable_name>)
   GDB: print <variable_name>  (or p <variable_name>)

7. To let the program finish running after a breakpoint:
   LLDB: continue (or c)
   GDB: continue (or c)

8. To exit the debugger:
   LLDB: quit (or q)
   GDB: quit (or q)

TPS 3
======

1. Four variables were declared: x, y, *px, *py.
   Two of them are pointers: px and py (both are int pointers).

2. The values of x, y, and arr[0] will be undefined/garbage values because they
   are not initialized. Local variables in C are not automatically initialized
   and contain whatever data was previously in that memory location.

3. Initialize them explicitly:
   int x = 0, y = 0;
   And use a loop to set each element of arr to 0:
   for (int i = 0; i < 10; i++) arr[i] = 0;

4. Use the %p format specifier with the address-of operator (&):
   printf("Address of x: %p\n", (void *)&x);
   printf("Address of y: %p\n", (void *)&y);

5. Set px = &x and py = &y.
   The value of px (printed with %p) equals the address of x.
   The value of py (printed with %p) equals the address of y.
   Pointers themselves also have addresses (using &px, &py).

6. Use pointer arithmetic with the dereference operator:
   for (i = 0; i < 10; i++) printf("%d ", *(arr + i));
   This uses arr as a pointer and adds offset i to access each element.

7. arr does point to the address of arr[0].
   Printing arr and &arr[0] with %p gives the same address, confirming that
   the array name acts as a pointer to the first element.

8. &arr gives the same numerical address as arr, but they have different types.
   arr is of type int* (pointer to int), while &arr is of type int(*)[10]
   (pointer to an array of 10 ints). They point to the same memory location
   but pointer arithmetic on them would differ. This makes sense because the
   array starts at its own address.
